Installation

Installation locale

Mettez en place l'environnement de développement NutriCam en 10 minutes.

1. Cloner le dépôt
bash
# Cloner le repo NutriCam
git clone https://github.com/nutricam/nutricam-app.git
cd nutricam-app

# Installer les dépendances
npm install

# Ou avec Yarn
yarn install
2. Variables d'environnement
.env.local
# ── BASE DE DONNÉES ──
DATABASE_URL="postgresql://nutricam:password@localhost:5432/nutricam_dev"
REDIS_URL="redis://localhost:6379"

# ── AUTHENTIFICATION ──
JWT_SECRET="your-super-secret-jwt-key-min-32-chars"
JWT_EXPIRES_IN="7d"
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="your-nextauth-secret"

# ── IA (ANTHROPIC CLAUDE) ──
ANTHROPIC_API_KEY="sk-ant-api03-..."
AI_MODEL="claude-sonnet-4-5"

# ── PAIEMENTS ──
CINETPAY_API_KEY="your-cinetpay-key"
CINETPAY_SITE_ID="your-site-id"
STRIPE_SECRET_KEY="sk_test_..."
STRIPE_WEBHOOK_SECRET="whsec_..."

# ── STOCKAGE FICHIERS ──
AWS_ACCESS_KEY_ID="AKIA..."
AWS_SECRET_ACCESS_KEY="..."
AWS_S3_BUCKET="nutricam-assets"
AWS_REGION="af-south-1"

# ── EMAIL ──
SMTP_HOST="smtp.gmail.com"
SMTP_PORT="587"
SMTP_USER="pitchouduplece9@gmail.com"
SMTP_PASS="your-app-password"

# ── NOTIFICATIONS (FIREBASE) ──
FIREBASE_PROJECT_ID="nutricam-app"
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n..."
3. Base de données
bash
# Générer le client Prisma
npx prisma generate

# Appliquer les migrations
npx prisma migrate dev --name init

# Seed : injecter les 52 recettes camerounaises
npx prisma db seed

# Vérifier avec Prisma Studio
npx prisma studio
# → Ouvre http://localhost:5555
4. Démarrer le serveur de développement
bash
# Démarrer le frontend (port 3000)
npm run dev

# Dans un autre terminal : API backend (port 8080)
npm run api:dev

# Vérification : tous les services démarrés
✓ Next.js prêt sur http://localhost:3000
✓ API NutriCam sur http://localhost:8080
✓ PostgreSQL connecté (nutricam_dev)
✓ Redis connecté (localhost:6379)
⚠ Paiements en mode sandbox (CinetPay test)
Déploiement

Déploiement Web (Production)

Déployer NutriCam sur Vercel (frontend) et AWS Lambda (backend) pour une infrastructure optimisée Afrique.

ℹ️
Recommandation : utilisez la région af-south-1 (Cape Town) pour AWS et activez le CDN Cloudflare avec nœuds Lagos + Nairobi pour minimiser la latence en Afrique subsaharienne (<80ms).
Déploiement Frontend — Vercel
bash
# Installer Vercel CLI
npm install -g vercel

# Se connecter et déployer
vercel login
vercel --prod

# Configurer le domaine personnalisé
vercel domains add nutricam.cm
vercel domains add www.nutricam.cm

# Variables d'environnement production
vercel env add DATABASE_URL production
vercel env add ANTHROPIC_API_KEY production
vercel env add NEXTAUTH_SECRET production

✓ Déploiement réussi → https://nutricam.cm
✓ SSL automatique activé (Let's Encrypt)
Déploiement Backend — Docker + AWS ECS
bash
# Build image Docker
docker build -t nutricam-api -f Dockerfile.prod .

# Tag pour AWS ECR (Elastic Container Registry)
docker tag nutricam-api \
  123456789.dkr.ecr.af-south-1.amazonaws.com/nutricam-api:latest

# Push vers ECR
aws ecr get-login-password --region af-south-1 | \
  docker login --username AWS --password-stdin \
  123456789.dkr.ecr.af-south-1.amazonaws.com

docker push 123456789.dkr.ecr.af-south-1.amazonaws.com/nutricam-api:latest

# Déployer sur ECS
aws ecs update-service \
  --cluster nutricam-prod \
  --service nutricam-api \
  --force-new-deployment

✓ Service mis à jour → api.nutricam.cm
Services tiers

Configuration des paiements

NutriCam supporte MTN Mobile Money, Orange Money (via CinetPay) et les cartes bancaires (Stripe).

CinetPay (Mobile Money Afrique)
✅
CinetPay agrège MTN MoMo et Orange Money dans une seule API. Idéal pour le Cameroun, Côte d'Ivoire, Sénégal.
javascript
// lib/payments/cinetpay.js
const CinetPay = require('cinetpay-sdk');

const client = new CinetPay({
  apiKey: process.env.CINETPAY_API_KEY,
  siteId: process.env.CINETPAY_SITE_ID,
  currency: 'XAF',           // FCFA
  channels: 'MOBILE_MONEY',
});

// Initier un paiement MTN/Orange
async function initiatePayment(userId, plan) {
  const transaction = await client.initPayment({
    amount: plan === 'premium' ? 2500 : 5000,
    transaction_id: `TXN-${Date.now()}`,
    customer_phone_number: user.phone,
    customer_name: user.name,
    description: `NutriCam ${plan} - Abonnement mensuel`,
    notify_url: 'https://api.nutricam.cm/webhooks/payment',
    return_url: 'https://nutricam.cm/dashboard',
  });
  return transaction.payment_url;
}
Webhook de confirmation
javascript
// routes/webhooks.js
app.post('/webhooks/payment', async (req, res) => {
  const { cpm_trans_id, cpm_site_id, cpm_amount, cpm_payment_date,
          cpm_payment_status } = req.body;

  // Vérifier la signature CinetPay
  const valid = verifyCinetPaySignature(req.body);
  if (!valid) return res.status(401).json({ error: 'Invalid signature' });

  if (cpm_payment_status === 'ACCEPTED') {
    // Activer l'abonnement
    await db.subscription.create({
      userId, plan: 'premium',
      startDate: new Date(),
      endDate: addMonths(new Date(), 1),
    });
    // Envoyer email de confirmation
    await sendConfirmationEmail(user.email, cpm_amount);
  }
  res.json({ status: 'received' });
});

Configuration IA (Claude API)

NutriCam utilise Claude d'Anthropic pour le chatbot nutritionnel, la génération de plans repas et les recommandations personnalisées.

javascript
// lib/ai/nutritionCoach.js
const Anthropic = require('@anthropic-ai/sdk');

const client = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
});

const SYSTEM_PROMPT = `
Tu es le Coach NutriCam, un expert en nutrition africaine
spécialisé dans la gastronomie camerounaise. Tu connais
parfaitement les 200+ recettes de notre base de données
avec leurs valeurs nutritionnelles exactes.

Règles importantes :
- Toujours citer des plats camerounais en priorité
- Adapter les conseils au profil de l'utilisateur
- Ne jamais recommander de produits importés si une
  alternative locale équivalente existe
- Mentionner les valeurs caloriques précises (kcal)
- Répondre en français naturel et bienveillant
`;

async function askCoach(userMessage, userProfile) {
  const response = await client.messages.create({
    model: 'claude-sonnet-4-5',
    max_tokens: 1024,
    system: SYSTEM_PROMPT,
    messages: [
      {
        role: 'user',
        content: `
Profil utilisateur : ${JSON.stringify(userProfile)}
Question : ${userMessage}
        `
      }
    ]
  });
  return response.content[0].text;
}
